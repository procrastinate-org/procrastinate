13# A few things about signals and asyncio: 

14# - https://docs.python.org/3/library/asyncio-eventloop.html#unix-signals 

15# - If you don't use loop.add_signal_handler, your handler has no effect on the queue 

16# - asyncio signal primitives don't give you a way to get the current handler for a 

17# signal 

18# - If a signal has an asyncio handler, its "standard handler (installed through 

19# signal.signal) is ignored (you can't get it, you can't set one, it doesn't get 

20# called on the signal). 

21# - asyncio handlers receive no parameter 

22# This all mean we have to save the previous signals before installing ours, and for 

23# uninstalling, we need to remove the async handler and only then re-add the normal 

24# one. And hope that there was no previously set async handler.
